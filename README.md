У цій базі даних spices реалізовано структуру магазину спецій, що зберігає дані про клієнтів, категорії, товари, замовлення та їхній вміст. Таблиці Customer, SpiceCategory, Spice, SpiceOrder і SpiceOrderContent пов’язані між собою так, що клієнт може мати кілька замовлень, кожне замовлення містить кілька спецій, а кожна спеція належить до певної категорії. У коді наведено приклади DML-запитів для додавання, оновлення та видалення даних, а також DQL-запитів для фільтрації, сортування, агрегації, об’єднання таблиць і виконання підзапитів. Запити дозволяють підраховувати кількість спецій у категоріях, визначати сумарну вартість запасів, витрати клієнтів, виявляти VIP-користувачів і знаходити товари, які ще не були замовлені.
